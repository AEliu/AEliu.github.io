计算机网络自顶向下方法一书的读书笔记。

Web 和 HTTP

Web 按需操作，其应用层协议是超文本传输协议( HTTP )，该协议有客户程序和服务器程序实现，分别运行在不同的端系统，通过交换 HTTP 报文进行会话。Web 页面由对象组成，含有一个 HTML 基本文件及引用对象，基本对象通过对象的 URL 地址引用页面的其他对象。HTTP 在 TCP 上运行，客户端发起与服务器的 TCP 连接，然后两端的进程通过套接字接口访问 TCP。用户请求页面时，浏览器向服务器发出请求报文，服务器收到请求进行响应。服务器向客户发送被请求的文件，但是不存储客户的状态信息，是一个无状态协议。

非持续连接与持续连接

多次的请求/响应中，每一次是通过相同的 TCP 连接还是经一个单独的 TCP 连接发送？分别是持续连接和非持续连接。非持续连接每个对象的请求/响应都断开 TCP 连接，一个对象传输时间是 2 个 RTT 与服务器传输 HTML 文件的时间，RTT 是往返时间，是一个短分组从客户到服务器然后返回客户所花费的时间，TCP 三次握手的前两次耗费的时间占用一个 RTT。每个请求都要建立和维护一个连接，采用非持续连接给服务器增加了负担，且增加了时间。

HTTP 1.1 持续连接的情况下，服务器发送响应后保持 TCP 连接打开，在此客户/服务器之间后续报文通过相同的连接传送，一条连接一段时间未被利用，服务器关闭该连接。
